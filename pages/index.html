Features

  • Simple declarative syntax.
  • A ng-include replacement
  • Support Inline JS Inside template
  • Support Script Tags with Src Attribute
  • Supports delayed loading
  • Control over script execution with noscript Attribute
  • Zero dependencies
  • Works on IE9+,Chrome,Opera,Firefox and all mobile browsers
  • Simple to use
  • can do what ng-include dose but without angular
  • 3kb normal side
  • 2kb minified

Roadmap

  • Support for autorefresh
  • Support for nested partial tags
  • Support for autorefresh number of times

Simple example 

    <partial src="templates/header.html"></partial>

Example with onload method

     <partial src="templates/footer.html" onload="Footerloaded"></partial>

NoScript attribute example 

   
     <partial src="templates/header.html" noscript="true"></partial>

     <partial src="templates/header.html" noscript></partial>

More complex example with manual loading of pages with javascript

Loads the partial after a sepecified delay

         <partial src="templates/menu.html" loadafter="3000"></partial>

JAVASCRIPT

    $('#menu a').click(function (e) {
        e.preventDefault();
        var path = $(this).prop('href');
        var container = document.getElementById('view');
        partial.getPage(path, container, function (elem) {
            Prism.highlightAll();
        });
